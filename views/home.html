You have created your first mixdown project.

Check out the following helpful projects. They are all part of this boilerplate.

  • Mixdown Server: Server and entry point for web site or service. Not required for a CLI project.
  • Mixdown Plugins: Common plugins for websites.
  • Mixdown Pipelines: Common task pipelines for handling request. Pipelines are not required with mixdown, but they are very helpful for sequencing plugin actions. Think of them as Controllers if you like.
  • Node Pipeline: Lightweight task sequencing for handling complex async sequences of operations. Similar to async.waterfall, but with a more expressive inteface and more event emitter feedback.
  • Broadway Handlebars: Broadway / Mixdown plugin for using the Handlebars template engine. Contains factory support and a few enhancements to helpers. This will also work clientside as a plugin.
